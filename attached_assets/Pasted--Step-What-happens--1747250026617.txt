| Step                    | What happens                                                                                                                                           | Tech we use                                                            |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------- |
| **1. OAuth login**      | Creator clicks “Connect my channel.”                                                                                                                   | Google OAuth scope: `youtube.readonly` + `youtube.analytics.readonly`. |
| **2. Data pull**        | We call YouTube APIs to fetch:<br>• audience‑retention curve (100 time buckets)<br>• captions (or we’ll transcribe).                                   | YouTube Analytics & Data APIs.                                         |
| **3. Align**            | Match every retention bucket with the subtitles that play in that window.                                                                              | Simple Python merge.                                                   |
| **4. Find hotspots**    | Detect sharp drops/spikes (≥ X % in Y seconds).                                                                                                        | Heuristic rules.                                                       |
| **5. Generate insight** | For each hotspot we feed the transcript snippet + drop size into an LLM prompt: **“Suggest two likely reasons for this viewer behavior and one fix.”** | GPT‑3.5 Turbo (≈ \$0.004/video).                                       |
| **6. Return report**    | Web dashboard / email:<br>• chart + colored call‑outs<br>• bullet‑point explanations<br>• to‑do list for next edit.                                    | React + Supabase (or similar).                                         |
